Simple Chat App

Something that is crucial for all programmers is to be able to understand and work inside of other people's codebases. This is a skill that I personally want to improve at.

I was brought onto an existing project at my new internship for Bay Valley Tech - Free Code Academy. The project was an instant messaging web app, and a lot of people had already put a lot of time and effort into it. Since this was one of the first times I had the opportunity to work in a complex codebase I didn't help create, I wanted to be as knowledgeable as I could about the project's tech stack. One particular area I had no experience in initially was using Google's Firestore database for real-time messaging.

Because I wanted to better understand Firestore, I spent some time creating a very simple instant messaging web application using a Firestore database. It was a great opportunity to learn more about the technology, and I wanted to share my thoughts.

Key Insights from Using Firestore

  1. Real Time Synchronization: Firestore made it very easy to implement this feature. With a simple function call, the database would automatically update listening clients whenever relevant data was modified.
  2. Direct Frontend to Database Communication: This was hard to get used to, but Firestore allowed you to set up custom read and write permissions based on whether a user was authenticated, which meant exposing the API key on the frontend was safe under certain conditions.
  3. NoSQL: Firestore is a NoSQL database, meaning data is stored in collections and documents, not tables. Both SQL and NoSQL have their use cases, and while I liked the structuredness of SQL I could see how the flexibility of something like Firestore could be nice.
